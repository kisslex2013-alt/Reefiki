# CORE #004 - Проверка смысла после зелёных тестов

Зелёный build отвечает на важный, но узкий вопрос: не сломалось ли то, что уже умеют проверять тесты. В агентской работе этого мало. Задача могла быть не только про код, а про смысл: сохранить ограничение, закрыть исходный brief, не уйти в соседнюю тему и не выдать красивый отчёт вместо результата.

В REEFIKI для этого появился `verify-intent`: небольшой deterministic gate, который сравнивает brief, результат или артефакт, изменённые файлы и ожидаемые слова/пути. Его задача не заменить человека, а дать evidence: задача действительно попала в исходный смысл или просто выглядит зелёной.

## Граница

`verify-intent` не принимает решение за автора и не гарантирует качество продукта. Он только заставляет результат пройти ещё одну проверку: совпадает ли сделанное с тем, что было реально запрошено.
