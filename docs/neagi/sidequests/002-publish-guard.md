# SIDE QUEST #002 - Почему агенту нельзя коммитить всё подряд

Когда с проектом работает AI-агент, в папке почти всегда есть больше, чем одна текущая задача: черновики, локальные заметки, generated files, работа другого агента, приватные материалы и публичные документы. Если просто сказать "закоммить всё", в историю может попасть лишнее.

В REEFIKI для этого есть publish guard: перед коммитом агент показывает, какие файлы попадут в изменение, сверяет их с задачей и сначала делает dry-run публикации. Это не бюрократия, а способ сохранить доверие к repo, когда с ним работают несколько агентов и есть отдельная public snapshot.

## Граница

Publish guard не публикует сам и не заменяет решение автора. Он только заставляет агента показать scope, доказать staged files и пройти safe publish path.
