# SIDE QUEST #017 - TeamLead integration owner

Когда над одним проектом работают несколько агентов, параллельный вывод сам по себе не равен готовой работе. Нужен один владелец интеграции: он держит общий scope, принимает результаты, проверяет доказательства и не даёт разным веткам смешать несовместимые решения.

Для читателя Не AGI это важная часть темы про управляемую автономность: агентам можно отдавать независимые задачи, но финальная сборка должна проходить через понятный evidence gate, а не через доверие к красивому отчёту.

## Что можно посмотреть

- Основной артефакт: [LEADOPS](../../../docs/LEADOPS.md)
- Контекст работы с ветками и worktree: [Worktree lifecycle](../../WORKTREE_LIFECYCLE.md)

## Граница

REEFIKI не заменяет владельца решения: она помогает держать порядок, evidence и безопасную передачу результата.
