# SIDE QUEST #020 - Browser runtime smoke

Фронтенд может проходить сборку и тесты, но всё равно ломаться в браузере: не загрузился asset, упал запрос, перекрылся интерфейс, кнопка не реагирует, страница выглядит иначе на другом размере экрана.

Поэтому browser smoke - это короткая проверка видимого поведения там, где его увидит человек. Для Не AGI это пример простого принципа: если результат должен работать в браузере, агент обязан посмотреть браузер, а не только читать код.

## Что можно посмотреть

- Основной артефакт: [browser-runtime-smoke-gate](../../../skills/base/browser-runtime-smoke-gate/SKILL.md)

## Граница

Такая проверка не делает полноценный QA-релиз, но быстро ловит ошибки, которые нельзя надёжно вывести из текста или build log.
