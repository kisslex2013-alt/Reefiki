# Domain - reefiki-demo

## Topic

Public-safe examples for REEFIKI first-run, public snapshot safety, and release
verification.

## Typical Sources

- REEFIKI public README and install docs.
- Public demo and command reference docs.
- Public-safe release checklists.

## Key Entities

- REEFIKI public repository.
- Public snapshot.
- Demo project.
- Release verification gate.

## Vocabulary

- "public snapshot" means the filtered public Git history published from the
  private source repository.
- "demo project" means this `reefiki-demo` project, not a private user project.

## What To Capture

Capture reusable public onboarding and verification procedures. Do not capture
private project content.
